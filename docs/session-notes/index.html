SVG Visual Diagram System — Established for All Sessions 2026-06-17

What was done: Inline SVG diagrams were injected directly into Section 3 (concept explanation) of sessions 01 and 02. Each SVG immediately follows the paragraph or code block it illustrates. No build step, no external images — pure inline SVG inside the existing HTML.

SVG style rules (apply to all future sessions):

  • Dark background: #111111
  • White stroke lines: stroke="white" stroke-width="1.5"
  • Pink accents: #ff5ca8 — used for highlights, notes, key labels, new elements
  • Font: font-family="monospace" throughout
  • Sketch feel: slight transform="rotate(Xdeg)" on boxes, stroke-linecap="round", stroke-linejoin="round"
  • Pink note boxes: <rect> with stroke="#ff5ca8" and fill="none" containing a label
  • Arrows: <path> or <line> + <polygon> arrowhead, filled matching stroke colour
  • Typical size: 480px wide, 155–185px tall
  • Wrapper: just the raw <svg> tag with style="display:block;margin:16px 0;"
  • Comment before each: <!-- SVG: concept name -->

Decision rule for where to insert: Insert a diagram after any code block or paragraph in Section 3 that introduces an abstract concept that a geometric shape, arrow, or container can make concrete. Skip paragraphs that are already obvious from the code example alone. Typical triggers: a new data structure, a before/after state change, a flow between two things, a comparison of two approaches, or a nesting relationship.
