Array map() and filter()

These are the two most important array methods in React. .map() is literally how every list in JSX gets rendered. .filter() is how search works. Master both here, and Sessions 14 and 21 will feel obvious.

2. Pre-Coding Quiz

Answer before reading the concept section. Need 4/5 to proceed to the lab.

Question 1 of 5

You call [1, 2, 3].map(x => x * 10). What is the result?

.map() always returns a new array of the same length. It never mutates the original. The callback runs once per item and its return value becomes the item in the new array.
Question 2 of 5

The callback passed to .filter() must return what type of value?

The filter callback is a predicate — it decides yes or no. Any truthy value (not just true) keeps the item. Any falsy value removes it. The item itself is not transformed — only included or excluded.
Question 3 of 5

You have const nums = [1, 2, 3, 4, 5];
What does nums.filter(n => n > 3) return?

.filter() returns a new array containing only items for which the callback returned truthy. Items 4 and 5 pass the n > 3 test.
Question 4 of 5

What is the key difference between .map() and .forEach()?

This is critical. .forEach() runs the callback for side effects and returns nothing (undefined). You cannot store its result. .map() collects the return value of each callback into a new array. In React JSX, you cannot use .forEach() to render a list because it returns nothing.
Question 5 of 5

You chain: countries.filter(c => c.region === "Americas").map(c => c.name)
Starting with 5 countries (2 in Americas, 3 elsewhere), what does this return?

Chaining works because both methods return arrays. .filter() produces a new array of 2 countries. .map() then transforms each of those 2 into just their name. Result: 2 names.

3. The Concept — map() and filter()

What map() does

.map() runs a function on every item in an array and collects the results into a brand new array of the same length. The original is never touched. 

const numbers = [1, 2, 3];

const doubled = numbers.map(function(n) {
  return n * 2;
});

console.log(doubled);  // [2, 4, 6]
console.log(numbers);  // [1, 2, 3] — unchanged

Think of it this way: .map() asks "for every item, what should it become?"

What filter() does

.filter() runs a test function on every item. Items that pass (truthy return) go into a new array. Items that fail (falsy return) are excluded. 

const countries = [
  { name: "Canada",  region: "Americas" },
  { name: "Japan",   region: "Asia" },
  { name: "Brazil",  region: "Americas" }
];

const americas = countries.filter(function(country) {
  return country.region === "Americas";
});

console.log(americas.length);  // 2
console.log(countries.length); // 3 — unchanged

Think of it this way: .filter() asks "for every item, should it be kept?"

The callback is just a function

The argument you pass to .map() and .filter() is just a normal JavaScript function. It receives the current item on each iteration. Whatever it returns determines the outcome. 

// These two are identical
const names1 = countries.map(function(country) { return country.name; });

function getName(country) { return country.name; }
const names2 = countries.map(getName);  // pass the function by reference

Chaining

Because both methods return arrays, you can chain them. Filter first (narrow the list), then map (transform the remaining items). 

const americaNames = countries
  .filter(function(c) { return c.region === "Americas"; })
  .map(function(c)    { return c.name; });

console.log(americaNames);  // ["Canada", "Brazil"]

map() vs forEach() — why this matters in React

// forEach — returns undefined. Cannot use in JSX.
const result1 = [1,2,3].forEach(n => n * 2);
console.log(result1);  // undefined

// map — returns array. Can embed in JSX.
const result2 = [1,2,3].map(n => n * 2);
console.log(result2);  // [2, 4, 6]
React connection: Inside JSX you will write: {'{'} countries.map(c => <CountryCard key={'{c.id}'} name={'{c.name}'} />) {'}'} This only works because .map() returns an array of JSX elements. .forEach() would return nothing and the screen would be blank.

What happens when no items match filter?

const result = countries.filter(c => c.region === "Antarctica");
console.log(result);        // []  — empty array, not null, not undefined
console.log(result.length); // 0

This matters in React: an empty array renders nothing, which is the correct behaviour for "no results found."

Use filter() for large populations

const bigCountries = countries.filter(function(country) {
  return country.population > 100000000;
});
console.log("Big countries:", bigCountries.map(function(c) { return c.name; }));

Notice you used .map() inside the console.log. Chaining inline.

6

Chain filter and map

const americaNames = countries
  .filter(function(c) { return c.region === "Americas"; })
  .map(function(c)    { return c.name; });

7

Prove originals are untouched

console.log("Original still has", countries.length, "items");
console.log("First item still:", countries[0].name);
8

Try forEach — see that it returns nothing

const forEachResult = countries.forEach(function(c) {
  return c.name;  // this return does nothing useful
});
console.log("forEach returned:", forEachResult);  // undefined

This is why React uses .map() for rendering, never .forEach().
