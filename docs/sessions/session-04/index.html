Arrow Functions

Every React codebase is written almost entirely in arrow functions. Every .map() callback, every component, every event handler. This session converts your understanding of functions into the syntax you will actually use.

Estimated time: 35–40 minutes

  • Explain what "implicit return" means and when curly braces break it
  • Understand why arrow functions handle this differently (enough to not be confused by it)
  • Rewrite all callbacks from Sessions 02 and 03 using arrow function syntax

2. Pre-Coding Quiz

Answer before reading the concept section. Need 4/5 to proceed.

Question 1 of 5

Which arrow function correctly doubles a number?

Without curly braces, the expression to the right of => is automatically returned. This is called implicit return. Option A has curly braces but no return keyword — it returns undefined. Option B works but the curly braces are unnecessary for a single expression.
Question 2 of 5

You want to implicitly return an object literal from an arrow function. Which is correct?

Curly braces are ambiguous — JavaScript sees them as a function body, not an object literal. Wrap the object in parentheses to signal "this is an expression, not a block".
Question 3 of 5

Which is valid syntax for an arrow function with no parameters?

Empty parentheses () are required when there are no parameters. You cannot omit them entirely.
Question 4 of 5

const greet = name => "Hello " + name;
What does greet("Canada") return?

Single parameter without parentheses is valid. No curly braces means implicit return. The expression "Hello " + name is evaluated and returned automatically.
Question 5 of 5

Arrow functions and regular functions handle this differently. Which statement is correct?

Arrow functions do not have their own this. They capture this from the lexical scope where they were defined. This is why React uses arrow functions for event handlers — they don't accidentally rebind this. In function components (which we use), this matters less, but it's the correct mental model.

3. The Concept — Arrow Functions

From regular function to arrow function

An arrow function is a shorter syntax for writing a function. These three are equivalent:

// Regular function declaration
function double(n) {
  return n * 2;
}

// Regular function expression
const double = function(n) {
  return n * 2;
};

// Arrow function
const double = n => n * 2;

Syntax rules

SituationSyntaxExample
No parameters() =>() => "hello"
One parameterx => or (x) =>n => n * 2
Two+ parameters(x, y) =>(a, b) => a + b
Single expressionNo braces — implicit returnn => n * 2
Multiple statementsCurly braces + explicit returnn => { const x = n * 2; return x; }
Return an objectWrap object in parensn => ({ value: n })

Implicit return — the most important rule

// With curly braces — you MUST write return
const add = (a, b) => {
  return a + b;
};

// Without curly braces — return is implicit
const add = (a, b) => a + b;

// Common mistake: curly braces with no return — silently returns undefined
const broken = (a, b) => {
  a + b;   // no return! This evaluates a+b and discards the result
};
console.log(broken(1, 2));  // undefined

Returning an object — the parenthesis trick

// WRONG — JavaScript reads { as start of function body
const makeObj = n => { value: n };  // returns undefined

// CORRECT — parentheses tell JavaScript "this is an expression"
const makeObj = n => ({ value: n }); // returns { value: 5 } when called with 5

With .map() and .filter() — this is how React code looks

const countries = [
  { id: 1, name: "Canada",  region: "Americas", population: 38000000 },
  { id: 2, name: "Japan",   region: "Asia",      population: 125000000 },
  { id: 3, name: "Brazil",  region: "Americas",  population: 215000000 }
];

// Before (regular function)
const names = countries.map(function(c) { return c.name; });

// After (arrow function — this is what React codebases look like)
const names = countries.map(c => c.name);

// Filter
const americas = countries.filter(c => c.region === "Americas");

// Chained
const americaNames = countries
  .filter(c => c.region === "Americas")
  .map(c => c.name);

Arrow functions and this

This is a subtle but important difference. Regular functions define their own this based on how they are called. Arrow functions capture this from the surrounding scope. 

const obj = {
  name: "Canada",
  greetRegular: function() { console.log(this.name); }, // "Canada" — this = obj
  greetArrow:   () =>       { console.log(this.name); }  // undefined — this = outer scope (not obj)
};

obj.greetRegular(); // "Canada"
obj.greetArrow();   // undefined (or global object's name)
React context: In React function components, you won't use this at all. The this difference matters more in class components (which we don't use) and in certain callback patterns. What matters now: know the rule exists, know arrow functions don't have their own this.

Note: formatCountry is defined once and passed by reference to .map(). The parentheses after formatCountry are absent — you are passing the function itself, not calling it.
