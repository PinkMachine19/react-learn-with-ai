3. The Concept — JavaScript First, Then React

JavaScript: map builds a new array

const countries = [
  { id: 1, name: "Canada", region: "Americas" },
  { id: 2, name: "Japan", region: "Asia" },
];

const names = countries.map(function (country) {
  return country.name;
});
// names → ["Canada", "Japan"]

Same length, transformed values. For React, the "transform" is JSX instead of strings.

React: map inside JSX

import countries from '../data/countries';
import CountryCard from './CountryCard';

function CountryList() {
  if (countries.length === 0) {
    return <p className="empty-message">No countries found</p>;
  }

  return (
    <section className="country-list">
      {countries.map(function (country) {
        return (
          <CountryCard
            name={country.name}
            region={country.region}
          />
        );
      })}
    </section>
  );
}

JSX can render an array of elements. React flattens them as siblings inside the section. Keys come in Session 15 — omitting them works for static lists but triggers warnings once lists change.

Arrow function implicit return

{countries.map(country => (
  <CountryCard name={country.name} region={country.region} />
))}

Parentheses after => mean implicit return. Curly braces without return give undefined.

Common mistakes

// MISTAKE: for loop inside JSX — use map instead
{ for (const c of countries) { … } }  // invalid

// MISTAKE: missing return in block body
countries.map(country => {
  <CountryCard name={country.name} />  // undefined each time
})

// MISTAKE: wrong import path
import countries from './countries';  // file is in src/data/

10. What Breaks If This Code Is Removed?

  • No map: Manual indexing breaks when array length changes.
  • No return in callback: Blank list — array of undefined.
  • Wrong import path: Build failure or empty module.
  • Removed empty check: If the data file is empty, map renders zero cards with no message (Session 13 regression).
