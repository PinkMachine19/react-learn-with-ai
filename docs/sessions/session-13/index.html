Layer 2 Session 13 React Fundamentals

Conditional Rendering

Not every render shows the same UI. Today you learn three JavaScript patterns — ternary, && short-circuit, and early return — and apply them in CountryList to show No countries found when the list is empty.

Estimated time: 35–40 minutes

1. Learning Objectives

  • Use the ternary operator: condition ? valueA : valueB
  • Use && short-circuit: condition && value
  • Use early return before the main JSX when a whole branch differs
  • Explain why if statements cannot sit directly inside JSX
  • Show "No countries found" when a countries array has length 0

2. Pre-Coding Quiz

Early return keeps the main return uncluttered when one guard case dominates.

3. The Concept — JavaScript First, Then React

JavaScript: ternary operator

An expression that picks one of two values:

const label = countries.length === 0 ? "No countries" : "Browse countries";

// same idea as:
let label;
if (countries.length === 0) {
  label = "No countries";
} else {
  label = "Browse countries";
}

Ternary returns a value — that is why it works inside JSX curly braces.

JavaScript: && short-circuit

const isLoggedIn = true;
isLoggedIn && console.log("Welcome");  // runs log

const count = 0;
count && console.log("Has items");  // does not log — 0 is falsy

In JSX, use && when you want "show this element or nothing":

{countries.length > 0 && <p>Pick a country</p>}
Pitfall: {count && <Badge />} when count is 0 renders the number 0 on screen (0 is falsy but React renders numbers). Prefer count > 0 && … or ternary.

JavaScript: early return in a function

function CountryList() {
  const countries = [];

  if (countries.length === 0) {
    return <p>No countries found</p>;
  }

  return (
    <section>
      <CountryCard name="Canada" region="Americas" />
    </section>
  );
}

The if is a normal JavaScript statement in the function body — not inside JSX.

React: three patterns in CountryList

// Pattern A — early return (recommended for empty list)
if (countries.length === 0) {
  return <p className="empty-message">No countries found</p>;
}

// Pattern B — ternary inside JSX (Fragment shorthand from Session 09)
return (
  <section>
    {countries.length === 0
      ? <p>No countries found</p>
      : (
        <div className="country-list">
          <CountryCard name="Canada" region="Americas" />
          <CountryCard name="Japan" region="Asia" />
        </div>
      )}
  </section>
);

// Pattern C — && for optional extras (not the main empty/list split)
{countries.length > 0 && <p>{countries.length} countries</p>}

10. What Breaks If This Code Is Removed?

  • No empty check: Empty array still tries to render countries[0] — undefined props or blank cards.
  • if inside JSX: Build error — statements not allowed in expressions.
  • Wrong && condition: Accidental 0 rendered on screen.
