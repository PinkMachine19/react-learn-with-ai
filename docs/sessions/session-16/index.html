Layer 3 Session 16 State

What Is State?

Layer 2 built a static country list — props flow down, UI matches data at render time, nothing changes after mount. Layer 3 asks: what happens when the user clicks, types, or toggles something and the screen must update? Today is concept only — no new app code. Session 17 introduces useState.

Estimated time: 35–40 minutes · No coding lab today

1. Learning Objectives

  • Distinguish props (passed in, read-only) from state (owned internally, can change)
  • Explain that updating state triggers a re-render of that component
  • Connect JavaScript variables to why ordinary local variables are not enough in React
  • Describe where state will live in the country app (preview: counters, search, filters)
  • Complete pre/post quizzes ≥ 80% before writing useState code in Session 17

3. The Concept — JavaScript First, Then React

JavaScript: values that change over time

let clicks = 0;

function onButtonPress() {
  clicks = clicks + 1;
  console.log(clicks);  // 1, 2, 3… in the console
}

The variable updates in memory. But a plain web page won't redraw a label unless something tells the DOM to update. React's state system connects "value changed" to "call component again and refresh UI."

Example in country appname, region on CountryCardClick count, search text, favorite toggle (later)

Re-render mental model

  1. User clicks a button (Session 17 lab wires a simple onClick for the counter; Session 18 goes deeper on events).
  2. The handler calls the state setter from useState (Session 17).
  3. React schedules a re-render of that component.
  4. The component function runs again — new JSX reflects new state.
  5. React diffs against previous output (Session 15 reconciliation) and updates the DOM.

Same function, second call — different return value because state changed. Props from parent may be the same; state values are not.

  • No event handlers — Session 18

10. What Breaks If This Concept Is Ignored?

  • Using only props for everything: Parent must re-render for every tiny interaction — awkward data flow.
  • Using only local variables: Clicks and typing appear to do nothing on screen.
  • Mutating props or shared imports: Breaks one-way flow; bugs hard to trace.
  • Skipping this session and jumping to useState: Syntax without mental model — confusion on when to use state at all.
