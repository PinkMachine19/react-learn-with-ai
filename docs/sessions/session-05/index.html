Layer 1 Session 05 JavaScript Foundations

Destructuring and Spread

These two features are how React codebases handle props and state every single day. Destructuring lets you pull values out of objects cleanly. Spread lets you copy or merge objects without mutation. Together they are the foundation of immutable state updates.

Estimated time: 35–40 minutes

1. Learning Objective

  • Destructure properties from an object into local variables
  • Destructure function parameters directly in the parameter list
  • Use the spread operator to copy an object
  • Use spread to create a new object with one property changed
  • Use spread to merge two objects
  • Explain why spread creates a shallow copy and what that means
  • Connect these patterns directly to how React props and state updates work

2. Pre-Coding Quiz

Answer before reading the concept. Need 4/5 to proceed.

Question 1 of 5

const country = { name: "Canada", region: "Americas" };
const { name, region } = country;
What is the value of name?

Destructuring extracts properties by name. const { name } = country is exactly the same as const name = country.name, just shorter.
Question 2 of 5

const original = { x: 1, y: 2 };
const copy = { ...original };
copy.x = 99;
What is original.x?

Spread creates a new object with the same top-level properties copied. Mutating a property on the copy does not affect the original — unlike const copy = original which shares the reference.
Question 3 of 5

You want to create a new object that is identical to country but with a different name. Which is correct?

Option A mutates the original (same reference). Option B: Object.copy doesn't exist. Option C: not valid syntax. Option D is the correct React pattern: spread all existing properties, then override specific ones. Later properties win when keys conflict.
Question 4 of 5

You have a function function show(country) { console.log(country.name); }
Which is the destructured parameter equivalent?

You can destructure directly in the parameter list. ({ name }) receives the same object but immediately extracts name. This is exactly how React components receive props: const CountryCard = ({ name, region }) => ...
Question 5 of 5

const a = { x: 1, nested: { y: 2 } };
const b = { ...a };
b.nested.y = 99;
What is a.nested.y?

Spread creates a shallow copy. Top-level primitive values are copied. But nested objects are still shared — only the reference to the nested object is copied, not the nested object itself. This is the shallow vs deep copy distinction.

3. The Concept — Destructuring and Spread

Object destructuring

const country = { name: "Canada", region: "Americas", population: 38000000 };

// Without destructuring
const name       = country.name;
const region     = country.region;
const population = country.population;

// With destructuring — same result, less repetition
const { name, region, population } = country;

Rename while destructuring

const { name: countryName, region: countryRegion } = country;
console.log(countryName);   // "Canada"
console.log(countryRegion); // "Americas"

Default values

const { name, capital = "Unknown" } = country;
console.log(capital); // "Unknown" — property didn't exist, default used

Destructuring function parameters — the React pattern

// Regular parameter
const showCountry = (country) => {
  console.log(country.name, country.region);
};

// Destructured parameter — this is how React components receive props
const showCountry = ({ name, region }) => {
  console.log(name, region);
};

showCountry({ name: "Canada", region: "Americas" }); // same result
React connection: When you write a component like const CountryCard = ({ name, region }) => ..., you are destructuring the props object in the parameter. React passes one object; you pull out the specific properties you need.

Spread operator — copying objects

const original = { name: "Canada", region: "Americas" };

// Spread: copies all key-value pairs into a new object
const copy = { ...original };

copy.name = "Japan";          // mutate the copy
console.log(original.name);  // "Canada" — original untouched
console.log(copy.name);      // "Japan"

Spread to update one property — the React state pattern

const country = { name: "Canada", region: "Americas", population: 38000000 };

// Create a new object with updated population — original unchanged
const updated = { ...country, population: 40000000 };

console.log(country.population); // 38000000 — unchanged
console.log(updated.population); // 40000000

// Later properties override earlier ones when keys conflict
const renamed = { ...country, name: "New Canada", region: "North" };
console.log(renamed);
// { name: "New Canada", region: "North", population: 38000000 }
Why this matters for React state: In Layer 3 you will update state like this:
setCountry({ ...country, population: 40000000 })
This creates a new object (new reference) so React detects the change and re-renders. If you wrote country.population = 40000000 instead, the reference wouldn't change and React would not re-render.

Spread arrays

const arr = [1, 2, 3];
const copy = [...arr];          // new array, same contents
const appended = [...arr, 4];   // new array with 4 added at end
const prepended = [0, ...arr];  // new array with 0 at start

console.log(arr);        // [1, 2, 3] — all unchanged
console.log(appended);   // [1, 2, 3, 4]
console.log(prepended);  // [0, 1, 2, 3]

This is the correct way to add an item to a React state array — instead of .push().

Shallow copy — the limitation

const a = { x: 1, nested: { y: 2 } };
const b = { ...a };

// Top-level: copied correctly
b.x = 99;
console.log(a.x); // 1 — safe

// Nested: still shared!
b.nested.y = 99;
console.log(a.nested.y); // 99 — a.nested was not copied, only the reference to it

Spread is shallow. Only the top level is copied. Nested objects share the same reference. For the country data we're working with (flat objects), this is fine.

4. Lab

Lab objective: Add destructuring and spread patterns to src/countries.js. Rewrite arrow function parameters to destructure. Create modified copies of countries with spread.
1

Destructure a country

const { name, region, population } = countries[0];
console.log(name, "—", region, "—", population.toLocaleString());
2

Rewrite map callback to destructure the parameter

// Before
const names = countries.map(c => c.name);

// After — destructure in the parameter
const names = countries.map(({ name }) => name);

console.log(names); // same result
3

More complex destructured callback

const summaries = countries.map(({ name, region, population }) =>
  name + " (" + region + ") — " + population.toLocaleString() + " people"
);
console.log(summaries);
4

Copy a country with spread

const canada = countries[0];
const canadaCopy = { ...canada };

canadaCopy.name = "New Canada";
console.log("Original:", canada.name);   // "Canada"
console.log("Copy:", canadaCopy.name);   // "New Canada"
5

Update one property with spread — the React state pattern

const updatedCanada = { ...canada, population: 40000000 };
console.log("Original population:", canada.population);   // 38000000
console.log("Updated population:", updatedCanada.population); // 40000000

This is exactly how you will update state in Session 17.

6

Spread arrays — simulate adding a country without mutation

const france = { id: 4, name: "France", region: "Europe", population: 68000000 };

// Add without push (non-mutating)
const expanded = [...countries, france];
console.log("Original:", countries.length);  // 3
console.log("Expanded:", expanded.length);   // 4

This is how you add an item to a React state array correctly.

7

Prove shallow copy limitation

const withLang = { ...canada, languages: { official: ["English", "French"] } };
const copy2 = { ...withLang };

copy2.languages.official.push("Spanish");
console.log("Original languages:", withLang.languages.official);
// ["English", "French", "Spanish"] — the nested object was shared!

You don't need to solve this. Just observe it and understand why it happens.

10. What Breaks If This Knowledge Is Missing?

  • Session 11 (Props): React destructures props constantly. Without this session, the syntax const CountryCard = ({ name, region }) => ... will be confusing.
  • Session 17 (useState): Every state update that modifies an object or array uses spread. Without spread, you will mutate state directly, and React will silently not re-render.
  • Session 21 (Lifting State): The entire pattern of updating shared state uses spread to create new objects.
