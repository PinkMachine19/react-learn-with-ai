Arrays and Iteration

Arrays are how you hold a list of things. In React, every list you render on screen starts as a JavaScript array. Understanding arrays — and how to loop through them — is the direct prerequisite for Session 14 where we render lists with .map().

Estimated time: 35–40 minutes
Prerequisite: Session 01 post-quiz score ≥ 80%

1. Learning Objective

By the end of this session you will be able to:

  • Create an array and access elements by index
  • Use .length to know how many items are in an array
  • Iterate over an array using a for loop and a for...of loop
  • Explain the difference between mutating and non-mutating array methods
  • Use .push(), .pop(), .includes(), and .indexOf()
  • Explain why React will care about whether you mutate an array or create a new one

2. Pre-Coding Quiz

Answer these before reading the concept section. Score honestly. Need 4/5 to proceed.

Question 1 of 5

You have const countries = ["Canada", "Japan", "Brazil"];
What does countries[1] return?

Arrays are zero-indexed. Index 0 is "Canada", index 1 is "Japan", index 2 is "Brazil". This is one of the most common off-by-one mistakes in programming.
Question 2 of 5

You write const a = ["X", "Y"]; then const b = a; then b.push("Z");
What is a.length after this?

Arrays are objects. Like all objects, const b = a copies the reference, not the contents. Both a and b point to the same array in memory, so b.push("Z") mutates the array that both variables reference. This is the same reference behaviour from Session 01, applied to arrays.
Question 3 of 5

Which of these loops correctly prints every item in const items = [10, 20, 30];?

Option A starts at index 1 (skips first item) and uses <= (accesses one past the end, returning undefined). Option B prints the index number, not the item. Option C uses <= so it reads items[3] which is undefined. Option D is correct: start at 0, stop before length, access items[i].
Question 4 of 5

.push() adds an item to an array. Which statement about it is true?

.push() is a mutating method. It changes the original array in-place and returns the new length of the array. This is important for React: if you store an array in state and push to it directly, React will not detect the change because the array reference hasn't changed. You'll learn the correct pattern in Layer 3.
Question 5 of 5

You have const list = ["a", "b", "c"];
What does list.indexOf("b") return?

.indexOf() returns the index of the first matching element, or -1 if not found. "b" is at index 1 (zero-indexed). -1 would be returned if "b" were not in the array. The pattern if (list.indexOf("b") !== -1) checks for existence — though .includes() is cleaner for that purpose.

3. The Concept — JavaScript Arrays

What is an array?

An array is an ordered list of values. Unlike an object (which stores values by key name), an array stores values by numeric position — called an index. 

// An object stores by name
const country = { name: "Canada", region: "Americas" };

// An array stores by position
const countries = ["Canada", "Japan", "Brazil"];
//                  index 0   index 1   index 2

Creating arrays and accessing elements

const countries = ["Canada", "Japan", "Brazil"];

// Access by index (zero-based)
console.log(countries[0]);  // "Canada"
console.log(countries[1]);  // "Japan"
console.log(countries[2]);  // "Brazil"
console.log(countries[3]);  // undefined — no index 3 exists

// Length: how many items
console.log(countries.length);  // 3

// Last item (works for any length)
console.log(countries[countries.length - 1]);  // "Brazil"
Common mistake: Arrays are zero-indexed. The last valid index is always length - 1. Accessing array[array.length] gives undefined, not an error.

Arrays are objects — reference behaviour applies

You learned in Session 01 that objects are passed and assigned by reference. Arrays are objects, so the exact same rules apply. 

const original = ["Canada", "Japan"];
const alias = original;        // NOT a copy — same reference

alias.push("Brazil");
console.log(original);  // ["Canada", "Japan", "Brazil"]
// original was mutated because alias points to the same array

Iterating — the for loop

A for loop runs a block of code a set number of times. It has three parts: initialise, condition, update. 

const countries = ["Canada", "Japan", "Brazil"];

for (let i = 0; i < countries.length; i++) {
  console.log(i, countries[i]);
}
// 0 "Canada"
// 1 "Japan"
// 2 "Brazil"

Read it as: start at i = 0, keep going while i is less than the length, after each iteration add 1 to i.

Iterating — the for...of loop

When you don't need the index, for...of is cleaner: 

for (const country of countries) {
  console.log(country);
}
// "Canada"
// "Japan"
// "Brazil"
React connection: In Session 03 you will learn .map() — a method that loops over an array and returns a new array of transformed values. .map() is how React renders lists. Everything in this session is the foundation for that.

Mutating vs non-mutating methods

This distinction matters enormously in React. Some array methods change the original array. Others return a new array and leave the original untouched.

Mutating — changes original
  • .push(item) — add to end
  • .pop() — remove from end
  • .unshift(item) — add to start
  • .shift() — remove from start
  • .splice(i, n) — remove/insert at index
  • .sort() — sort in place
  • .reverse() — reverse in place
Non-mutating — returns new array
  • .map(fn) — transform each item
  • .filter(fn) — keep matching items
  • .slice(i, j) — copy a portion
  • .concat(arr) — combine arrays
  • [...arr] — spread copy
  • .find(fn) — returns one item
  • .reduce(fn) — accumulate to one value

In Layer 3, when you learn useState, you will discover that React only re-renders when it sees a new reference. If you mutate an array in state, React sees the same reference and does nothing. You must return a new array. That is why this distinction matters right now — not just as trivia.

Useful array methods to know now

const countries = ["Canada", "Japan", "Brazil"];

// Check if item exists
console.log(countries.includes("Japan"));   // true
console.log(countries.includes("France"));  // false

// Find the position
console.log(countries.indexOf("Brazil"));   // 2
console.log(countries.indexOf("France"));   // -1  (not found)

// Add to end (mutating)
countries.push("France");
console.log(countries);  // ["Canada", "Japan", "Brazil", "France"]

// Remove from end (mutating)
const removed = countries.pop();
console.log(removed);    // "France"
console.log(countries);  // ["Canada", "Japan", "Brazil"]

// Copy a portion (non-mutating)
const first2 = countries.slice(0, 2);
console.log(first2);     // ["Canada", "Japan"]
console.log(countries);  // ["Canada", "Japan", "Brazil"] — unchanged

Arrays of objects — the pattern React uses

In real React applications, you almost never have an array of plain strings. You have an array of objects — one object per item to display. 

const countries = [
  { id: 1, name: "Canada",  region: "Americas" },
  { id: 2, name: "Japan",   region: "Asia" },
  { id: 3, name: "Brazil",  region: "Americas" }
];

// Access a specific country's name
console.log(countries[0].name);   // "Canada"
console.log(countries[1].region); // "Asia"

// Loop and access properties
for (const country of countries) {
  console.log(country.name, "-", country.region);
}
// Canada - Americas
// Japan - Asia
// Brazil - Americas

The id field on each object is not accidental. In Session 15, you will learn that React needs a unique key for every item in a rendered list. That key comes from here.

6

Use .includes() and .indexOf()

const names = ["Canada", "Japan", "Brazil"];

console.log("Includes Japan?", names.includes("Japan"));   // true
console.log("Includes France?", names.includes("France")); // false
console.log("Index of Brazil:", names.indexOf("Brazil"));  // 2
console.log("Index of France:", names.indexOf("France"));  // -1
7

Prove mutation: push and pop

console.log("Before push:", countries.length); // 3

countries.push({ id: 4, name: "France", region: "Europe", population: 68000000 });
console.log("After push:", countries.length);  // 4

const removed = countries.pop();
console.log("Removed:", removed.name);         // "France"
console.log("After pop:", countries.length);   // 3

Notice that .push() and .pop() changed countries directly. The array reference is the same — only the contents changed.

8

Prove non-mutation: slice

const americasOnly = countries.filter(function(country) {
  return country.region === "Americas";
});

console.log("Filtered:", americasOnly.length);   // 2
console.log("Original:", countries.length);      // 3 — unchanged

.filter() is a preview of Session 03. Notice: it does not change countries. It returns a brand new array. This is the pattern React requires for state updates.

10. What Breaks If This Knowledge Is Missing?

  • Off-by-one errors: Starting a for loop at 1 instead of 0, or using <= instead of <, skips the first item or reads one past the end (returning undefined). These are silent — no error thrown.
  • React state mutations (Layer 3): Using .push() or .pop() directly on a state array causes the most common React beginner bug: the UI doesn't update even though the data changed. React compares references, not contents.
  • Session 03 — map/filter: Those methods take a function argument and return new arrays. If you don't understand what "returns a new array" means, the Session 03 labs will be confusing.
  • Session 15 — Keys: React's key prop must uniquely identify each list item. It comes from the id field in your array objects. Duplicate or missing ids cause subtle rendering bugs.
