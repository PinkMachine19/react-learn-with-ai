3. The Concept — JavaScript First, Then React

JavaScript: nesting structure

HTML nests tags: a <ul> contains many <li> elements. JavaScript functions can call other functions and combine their results: 

function listItem(label) {
  return "<li>" + label + "</li>";
}

function countryList() {
  return "<ul>" +
    listItem("Canada") +
    listItem("Japan") +
  "</ul>";
}

countryList does not rebuild what a list item looks like — it uses listItem. That is composition at the function level.

React: components nest like HTML

A component's return value can include other components. React runs each function and assembles the tree: 

// App.jsx — high level
function App() {
  return (
    <div>
      <h1>Countries</h1>
      <CountryList />
    </div>
  );
}

// CountryList.jsx — owns the list
import CountryCard from './CountryCard';

function CountryList() {
  return (
    <section className="country-list">
      <CountryCard name="Canada" region="Americas" />
      <CountryCard name="Japan" region="Asia" />
      <CountryCard name="Brazil" region="Americas" />
    </section>
  );
}

export default CountryList;

App no longer knows how many countries exist or how cards are laid out — it only knows "show the list." Props still flow CountryList → CountryCard.

Folder note: The curriculum uses src/components/ (plural). If your Session 10 file lives in src/component/ (singular), move it to src/components/CountryCard.jsx and update imports before this lab.

Common mistakes

// MISTAKE: lowercase tag — React treats it as an HTML element, not your component
return <countrycard name="Canada" />;

// MISTAKE: forgetting import in CountryList.jsx
return <CountryCard />;  // ReferenceError — CountryCard not defined

// MISTAKE: multiple root elements without a wrapper
return (
  <CountryCard />
  <CountryCard />  // syntax error — need one parent (Session 09: div or <></>)
);

10. What Breaks If This Code Is Removed?

  • No CountryList: App grows cluttered; list layout and card data mix with page chrome.
  • No import in CountryList: CountryCard is undefined — blank screen or error overlay.
  • No wrapper element: Syntax error — JSX return must have one root (or a Fragment).
  • Cards stay in App: Lab incomplete — you did not practice composition.
