Layer 2 Session 09 React Fundamentals

JSX Deep Dive

JSX looks like HTML inside JavaScript, but it is not HTML. It is syntax sugar for React.createElement — a function call that returns a plain JavaScript object describing what should appear on screen. Today you learn the rules, write your first JSX with an embedded expression, and see what the compiler produces underneath.

Estimated time: 35–40 minutes  ·  Pre-quiz → Concept → Lab → Commit → Post-quiz

1. Learning Objectives

  • Explain that a JavaScript function call returns a value — and JSX is built on that idea
  • Describe what React.createElement returns and why JSX is syntax sugar for it
  • Apply JSX rules: one root element, camelCase attributes, className, self-closing tags
  • Embed a JavaScript expression inside JSX using curly braces {"{}"}
  • Contrast HTML attribute names with their JSX equivalents
  • Update App.jsx to render "Hello World" with a dynamic expression on screen

2. Pre-Coding Quiz

Answer before reading the concept. Need 4/5 to proceed.

Question 1 of 5

function sayHi() {
  console.log("hi");
}
const result = sayHi();
What is the value of result?

Functions only return a value when you use return. console.log prints to the console but does not return anything. Without an explicit return, the function returns undefined.
Question 2 of 5

What does React.createElement('h1', null, 'Hello') return?

createElement is a function call that returns a React element object — a description of what to render. React turns that object into real DOM later. It is not HTML text and not a live DOM node yet.
Question 3 of 5

How do you embed the result of 2 + 2 inside JSX text?

Curly braces in JSX mean "evaluate this JavaScript expression." Without braces, everything inside the tag is treated as plain text.
Question 4 of 5

Which is valid JSX for applying a CSS class?

class is a reserved word in JavaScript. JSX attributes become object properties, so React uses className instead of class.
Question 5 of 5

Can this be valid JSX returned from a component?

return (
  <h1>Hello</h1>
  <p>World</p>
);
A JSX expression must have exactly one root element. Two sibling elements need a wrapper (<div>) or a Fragment (<>…</>).

3. The Concept — JavaScript First, Then JSX

Step 1 — Functions return values

Before JSX makes sense, remember what you already know from Layer 1: a function call produces a return value. That value can be a string, a number, an object, or undefined if nothing is returned. 

function greet(name) {
  return "Hello, " + name;
}

const message = greet("World");  // message is "Hello, World"

function broken() {
  console.log("I print but don't return");
}
const nothing = broken();        // nothing is undefined

React components are functions too. Whatever they return is what React tries to put on screen. JSX is just a readable way to write that return value.

Step 2 — React.createElement is a function call

Under the hood, this JSX: 

const element = <h1 className="title">Hello World</h1>;

compiles to this JavaScript:

const element = React.createElement(
  'h1',
  { className: 'title' },
  'Hello World'
);

React.createElement takes three arguments: the tag name (or component), a props object, and children. It returns a plain object — a React element. Not HTML. Not a DOM node yet. Just a description React can work with.

Step 3 — JSX rules

  • One root element — a return must wrap siblings in one parent (or Fragment)
  • camelCase attributes — onClick, tabIndex, htmlFor
  • className not class — because props become a JS object
  • Expressions in {"{}"} — any valid JS expression evaluates inside JSX
  • Self-closing tags — <img />, <input />, <br />
  • Fragments (preview) — <>…</> groups siblings without an extra DOM node (details later)

HTML vs JSX — attribute differences

JSX attributes look like HTML but behave like JavaScript object keys. The table below lists the most common differences you will hit in this project.

HTMLJSXWhy
class="box"className="box"class is a reserved word in JavaScript
for="email"htmlFor="email"for is a reserved word in JavaScript
onclick="…"onClick={"{handler}"}camelCase; value is a JS function, not a string
tabindex="0"tabIndex={0}camelCase; numbers use braces without quotes
style="color:red"style={"{{ color: 'red' }}"}style is a JS object, not a CSS string (details later)
<img> (no close)<img />JSX requires explicit self-closing syntax

What the browser actually shows

JSX describes a tree of elements. React walks that tree and creates matching DOM nodes. Your lab today produces a simple tree: one heading with static text, plus a paragraph whose content comes from a JavaScript expression.

Layer 1 connection: Session 04 taught arrow functions — every React component is one. Session 06 taught imports — you will write import React from 'react' (or rely on the modern JSX transform). JSX expressions use the same variables and arithmetic you already know from Layer 1.

Common mistakes

// MISTAKE: class instead of className
<div class="container">  // syntax error or warning

// MISTAKE: double braces confusion — outer = JSX expression, inner = object
<div style={{ marginTop: 10 }}>  // correct (learn fully later)
<div style="{ marginTop: 10 }">   // wrong — string, not object

// MISTAKE: forgetting braces — renders literal text
<p>2 + 2</p>           // screen shows: 2 + 2
<p>{2 + 2}</p>          // screen shows: 4

// MISTAKE: two root siblings
return (
  <h1>Hi</h1>
  <p>Bye</p>
);  // syntax error — wrap in <div> or <></>

function App() {
  const countryName = "Canada";

    <div>

}

export default App;

8. Post-Coding Quiz

Question 1 of 5

A function with no return statement returns:

JavaScript functions without an explicit return always return undefined.
Question 2 of 5

JSX is best described as:

JSX compiles to React.createElement(...) calls. It is not native HTML and not a separate language.
Question 3 of 5

What renders on screen for <span>{"{countryName}"}</span> when countryName = "Japan"?

Curly braces evaluate the expression. The variable's value is rendered, not its name.
Question 4 of 5

Why must void elements like <img> be written as <img /> in JSX?

JSX follows XML-like rules. Every tag must be explicitly closed. <img> without /> is a syntax error.
Question 5 of 5

If you remove the curly braces from <p>{"{2 + 2}"}</p>, what appears on screen?

Without braces, JSX treats content as a string literal. The characters "2 + 2" display as text; no math runs.

10. What Breaks If This Code Is Removed?

  • Remove the root wrapper: Two sibling elements cause a compile error — the file will not build.
  • Remove curly braces from expressions: Variables and math render as literal text. The UI looks "broken" even though the app runs.
  • Use class instead of className: Styles silently fail or the build warns — elements lose their intended appearance.
  • Session 10 onward: Every component returns JSX. Without today's rules, every new component file will hit the same syntax errors repeatedly.
